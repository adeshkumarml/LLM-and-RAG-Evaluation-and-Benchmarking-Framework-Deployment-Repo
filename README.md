# LLM and RAG Evaluation & Benchmarking Framework
A RAG evalutaion and benchmarking framework for comparing baseline large language model outputs against retrieval-augmented generation using  retrieval metrics, semantic similarity, and LLM-as-judge evaluation.

## Links and Demo
Access the system *(v1)* from here: https://llmragbenchmarking.streamlit.app/<br>

## Overview
RAG systems are commonly used to improve LLM responses by grounding them in external context. However, evaluating whether retrieval actually improves response quality remains a challenging problem.<br>
This evaluation framework compares baseline LLM generations against retrieval-enhanced generations using multiple evaluation strategies. The system measures how effective the retrieval is, semantic similarity, exact match, and qualitative response quality through LLM-as-judge evaluation.<br>
Users can upload a corpus and evaluation dataset through an interactive Streamlit interface, run automated benchmarking pipelines, and analyze structured evaluation reports generated by the FastAPI backend.

## Features
- RAG evaluation pipeline, LLM vs RAG response benchmarking<br>
- Semantic retrieval using sentence-transformer embeddings<br>
- Exact match evaluation & semantic similarity scoring<br>
- Retrieval metrics: precision@k, recall@k, and mean-reciprocal rank<br>
- LLM-as-judge evaluation: completeness, correctness, and fluency<br>
- Interactive streamlit UI and FastAPI for inference<br>
- Aggregate evaluation metrics and sample-wise evaluation metrics<br>
- Downloadable JSON reports

## Tech-Stack
The following have been used:<br>
- Embedding Model: Sentence Transformer all-MiniLM-L6-v2<br>
- LLM Model: OpenAI gpt-4o-mini<br>
- Backend: FastAPI<br>
- Frontend: StreamlitUI<br>
- Deployment: Render, Streamlit Cloud<br>

## General Architecture
User uploads corpus(.txt) and dataset(.json) ➡️ Streamlit UI ➡️ FastAPI ➡️ Evaluator ➡️ Retrieval and generation ➡️ LLM judge and metrics<br>
➡️ .json results and dashboard

## Deployment Notes & Known Issues

- The current framework is designed specifically for QA tasks only.
- Initial requests may experience cold-start latency due to transformer model loading on Render's free-tier.
- Evaluation datasets' maximum sample size has been limited to 10 to maintain manageable inference latency and API usage.. 
- Larger datasets  may require longer processing times because generation and evaluation are currently performed sequentially.
- Retrieval metrics based on lexical overlap may report semantically correct retrievals when phrasing differs from the actual answers.
- Uploaded datasets are expected to follow the exact JSON schema format shown in Streamlit UI<br>
